Debrief

Vulnerability Explained

Critical Prototype Pollution (CVE Class: PP)

Prototype Pollution is a JavaScript-specific vulnerability where an attacker can inject properties into the global Object.prototype. Since every JavaScript object inherits from Object.prototype, any property added to the prototype becomes accessible on all objects — including freshly created empty ones.

In the Watson Protocol, the vault's access check was:

// The fatal check — reads inherited property, not own property if (({}).clearance === "OMEGA_7") { // grant access }

An empty object literal {} has no own clearance property. But if someone had previously poisoned Object.prototype.clearance = "OMEGA_7", every object in the process inherits it — and the check passes.

Attack Path Step-by-Step Reconstruction
  1. Read /v3/internal/notes.txt with the Watson auth header. The migration log mentions "ClientSideHydration" and a debug key PROTOTYPE_SYNC_ENABLET — hinting at the vulnerability class.
  2. Read /v3/internal/archive/naming_convention.json. It reveals the encoding scheme: hex_base64 with a reverse_stream_v2 format.
  3. Query the GraphQL endpoint with the auth header. The alpha_sector_logs response returns a reversed hex string in its logs field.
  4. Decode the signal: reverse the hex string → decode from hex → base64-decode the result → parse as JSON. The JSON reveals {"__proto__": ...} — confirmation of the attack vector.
  5. POST a crafted payload to /api/v1/hydrate: {"__proto__": {"clearance": "OMEGA_7"}}. Despite the naïve filter that replaces "__proto__", the endpoint calls _.merge(Object.prototype, data) directly — bypassing its own sanitization and polluting the prototype.
  6. GET /api/v1/vault/open. The check ({}).clearance === "OMEGA_7" now evaluates to true. The vault opens. The flag is captured.
Defence How to Fix It — Securing the Code

There are multiple layers where this vulnerability could have been prevented:

1. Never pass untrusted data to prototype-mutating functions:

// VULNERABLE: merges directly into Object.prototype _.merge(Object.prototype, data);
// SAFE: merge into a plain container object, never into prototypes const safeContainer = {}; _.merge(safeContainer, data); res.json({ status: "success", data: safeContainer });

2. Use Object.create(null) for objects that should have no prototype:

// No prototype chain — immune to pollution const safeObj = Object.create(null); // ({}).clearance will still be undefined even after pollution

3. Use hasOwnProperty for security-critical checks:

// VULNERABLE: reads inherited properties if (({}).clearance === "OMEGA_7") { ... }
// SAFE: only reads own properties const ctx = {}; if (Object.prototype.hasOwnProperty.call(ctx, 'clearance') && ctx.clearance === "OMEGA_7") { ... } // Or better yet: use explicit credential parameters, not object property checks

4. Freeze Object.prototype in production:

// Called once at application startup Object.freeze(Object.prototype); // Now no code — including libraries — can mutate the prototype

5. Validate and sanitize all JSON input deeply:

function sanitize(obj) { const DANGEROUS = ['__proto__', 'constructor', 'prototype']; if (typeof obj !== 'object' || !obj) return obj; for (const key of DANGEROUS) delete obj[key]; for (const val of Object.values(obj)) sanitize(val); return obj; } const data = sanitize(JSON.parse(req.body));

Implement all five layers — defence in depth ensures that even if one layer fails, the others hold. The Watson Protocol's mistake was trusting a partially-sanitized payload and then immediately passing raw data to a prototype-mutating function.
